## travel-companion: web dev 自动加载 fixture
- 给 `web/vite.config.ts` 加了一个 `apply: "serve"` 的 dev-only plugin `devFixturePlugin`：在每次页面加载时把 `fixtures/<FIXTURE>/data/trip.json` 内联进 `index.html` 的 `__TRIP_DATA__` 占位（正则匹配 `__TRIP_DATA_BEGIN__/END__` 注释）。
- 默认 fixture 是 `egypt-south`；可用 `FIXTURE=other-route npm run dev` 切换。
- `npm run build` 不触发该 plugin（apply:"serve"），所以生产构建仍然让 `__TRIP_DATA__` 保持 `null`，由 `scripts/inject.py` 在部署时注入真实 trip data，不破坏现有 pipeline。
- 在 fixture 目录里直接用 `python3 scripts/export_data.py --trip-root fixtures/egypt-south` 跑出 `data/trip.json`（21 实体, 17KB），dev server 才能直接读取。
- 验证：`cd web && npm run dev` 后访问 http://localhost:5173/，控制台日志 `[travel-companion] injecting fixture: egypt-south (17244 bytes)`，curl 出来的 HTML 里 `__TRIP_DATA__` 已经是合法 JSON，包含 21 entities + 4 recs + 10 turns。
- 同步更新了 `web/README.md`、`SKILL.md` Test fixtures 段、`fixtures/egypt-south/README.md` 中的 quick-start。
